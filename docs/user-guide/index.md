# Babysitter User Guide

Welcome to the Babysitter documentation. Babysitter is an AI-powered development workflow orchestrator that enables iterative quality convergence with human-in-the-loop approval through breakpoints.

## Learning Paths

Choose a path based on your role and goals.

### For Developers New to Babysitter

1. Read [What is Babysitter?](./features/quality-convergence.md) to understand the core concepts
2. Complete the [Getting Started](./getting-started/README.md) tutorial
3. Build your first project with [REST API Tutorial](./tutorials/beginner-rest-api.md)
4. Explore the [Glossary](./reference/glossary.md) for terminology

### For Experienced Developers

1. Quick setup via [Installation](./getting-started/installation.md)
2. Review [Process Definitions](./features/process-definitions.md) for customization
3. Learn [Breakpoints](./features/breakpoints.md) for approval workflows
4. Reference the [CLI](./reference/cli-reference.md) for automation

### For Technical Leads and Architects

1. Understand [Quality Convergence](./features/quality-convergence.md) philosophy
2. Review [Journal System](./features/journal-system.md) for audit compliance
3. Explore [Custom Process](./tutorials/intermediate-custom-process.md) for team workflows
4. Plan with [Multi-Phase Workflows](./tutorials/advanced-multi-phase.md)

### For DevOps and Automation Engineers

1. Install using [Quickstart](./getting-started/quickstart.md)
2. Master the [CLI Reference](./reference/cli-reference.md)
3. Configure via [Configuration Reference](./reference/configuration.md)
4. Automate with [Run Resumption](./features/run-resumption.md)
